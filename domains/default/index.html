Configuring Twine

As you may have noticed, Twine has generated a few configuration files. One among these, twine.yml, plays a large role in how your server behaves, so it's best to configure it before going into production.
Specifically, you should take a look at ip and port. ip and port should be pretty self-explanatory; they determine the IP/address and port to which the server should bind. To configure domains, take a look at domains.yml to get an example on how it works.
